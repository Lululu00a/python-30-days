# 练习 3 参考答案

## 为什么 Python 不需要 main 函数？

Python 是**解释型语言**，解释器从文件第一行开始**逐行执行**，遇到什么就执行什么；
而 C/C++ 是**编译型语言**，链接器需要一个固定入口符号 `main`，操作系统加载后跳转到这里执行。

## 优点
1. **快速验证想法**：写几行就能跑，REPL 也能直接交互，特别适合学习、做实验、写脚本
2. **降低入门门槛**：初学者不必先理解"函数""返回值"就能写出 `print("hi")`
3. **脚本灵活**：自动化运维、数据处理脚本通常很短，不需要繁琐的入口结构

## 缺点
1. **大型项目易混乱**：模块被 `import` 时，顶层代码会被立即执行，可能产生副作用
2. **入口不清晰**：阅读他人代码时，要靠经验判断"程序从哪里开始"
3. **测试不便**：顶层代码不能像函数那样被 mock 或单独调用

## 工程实践
正因如此，**项目级 Python 代码**仍然约定使用：
```python
if __name__ == "__main__":
    main()
```
这样模块既能被 `import` 复用，又能作为脚本直接执行——**鱼和熊掌兼得**。
